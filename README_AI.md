# README_AI

## WFS Status

- Current Status: `Candidate`
- 本システムはWellzFriend Standard策定のベースモデルである。
- 実運用で改善し、検証済みの知見だけを正式版WFSへ反映する。
- `Candidate`はWFS成熟度であり、アプリのライフサイクル状態とは別に管理する。

## 目的

弥生会計等の試算表から作成した正規化明細をDATAへ貼り付け、部門別の累計損益と前年同期累計を自動生成する。会社全体PLの代替ではなく、部門ごとの伸長・改善・縮小判断を支援する。

## 実装範囲

- 部門: れんこん加工、青果加工、共通
- 共通経費: 配賦せず独立表示
- 比較: 当期累計、前年同期累計
- 主画面: 部門別累計PL、売上成長率、営業利益増減、営業利益率
- KPI: 売上、粗利益、営業利益、粗利率、営業利益率
- グラフ: 売上推移、利益推移、部門別売上、部門別営業利益
- 提出資料: A4横1枚のEXECUTIVE_REPORT
- 成果物1 DASHBOARD: 日々の経営判断用。PC画面での利用を前提とし、印刷は必須としない。
- 成果物2 REPORT: 部門別累計PLと詳細分析用。A4印刷・PDFに対応する。
- 成果物3 EXECUTIVE_REPORT: 金融機関、税理士、役員会、経営会議、補助金申請への提出用経営サマリー。

## シート責務

| シート | 責務 | 通常編集 |
|---|---|---|
| README_AI | Sheets内の引継ぎ説明 | 保守時のみ |
| CHANGELOG | 更新履歴 | 更新時 |
| SETTINGS | 基本設定、部門、科目マッピング | 管理者 |
| DATA | 元明細 | 利用者 |
| CALC | 月次・部門別集計 | 禁止 |
| DASHBOARD | 部門別累計カード、前年同期比較、判断メモ、参考グラフ | メモのみ |
| REPORT | 部門別累計PL、前年同期累計比較、製造経費累計内訳 | 禁止 |
| EXECUTIVE_REPORT | 経営サマリー・提出資料 | 禁止 |

## 入力契約

DATAの1行は1会計明細。入力列は `計上日 / 部門 / 勘定科目 / 金額 / 摘要`。年月とPL区分は自動列。部門名と勘定科目はSETTINGSのマスタと完全一致させる。

## 計算

- 売上原価 = 材料費 + 労務費 + 製造経費
- 売上総利益 = 売上高 - 売上原価
- 営業利益 = 売上総利益 - 販管費
- 粗利率 = 売上総利益 ÷ 売上高
- 営業利益率 = 営業利益 ÷ 売上高
- 比較率 = (当月 - 比較月) ÷ 比較月
- 表示月は月初へ正規化して集計し、SETTINGSの日付が月初・月末のどちらでも同じ月として扱う。
- 製造原価の使用科目はSETTINGSの完全一致マスタで管理する。
- REPORTでは製造経費8科目の内訳を折りたたみ・展開できる。
- 当期累計はSETTINGSの会計年度開始月から表示月まで、前年同期累計はその12か月前の同期間を集計する。

## 参照方向

`DATA → CALC → REPORT / DASHBOARD / EXECUTIVE_REPORT` を唯一の参照方向とする。

- DATA: 入力明細と自動分類だけを保持する。
- CALC: 月次、累計、前年同期、部門比較、製造経費科目内訳を計算する唯一のシート。
- REPORT: CALCをXLOOKUPで取得して表示する。SUMIFSとDATA参照は禁止。
- DASHBOARD: REPORTとCALCのグラフ用出力を直接表示する。SUMIFS、XLOOKUP、DATA参照は禁止。
- EXECUTIVE_REPORT: CALCの計算済み結果だけをXLOOKUPで表示する。DATA参照と再集計は禁止。

REPORT、DASHBOARD、EXECUTIVE_REPORTはCALCから並列参照し、表示成果物同士を計算経路として直列接続しない。

## 変更手順

1. 目的と受入条件を更新する。
2. SETTINGSと入力契約への影響を確認する。
3. CALC、REPORT、DASHBOARD、EXECUTIVE_REPORTの順に変更する。
4. `tests/TEST_CASES.md`を実施する。
5. CHANGELOGを更新する。

## 禁止事項

- DATAからDASHBOARDへ直接依存を追加しない。
- CALCへ会社固有値を埋め込まない。
- 実データや秘密情報をGitHubへ保存しない。
- WFS候補を検証前に正式標準へ昇格しない。
